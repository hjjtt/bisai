# Triage Labels

This repo uses the default triage label vocabulary. Each label corresponds to a state in the issue triage state machine.

## Labels

| Label | State | Meaning |
|---|---|---|
| `needs-triage` | Evaluation | Maintainer needs to evaluate the issue |
| `needs-info` | Waiting | Waiting on reporter for more information |
| `ready-for-agent` | AFK-ready | Fully specified, an agent can pick it up with no human context |
| `ready-for-human` | Human needed | Requires human implementation or decision |
| `wontfix` | Rejected | Will not be actioned |

## Mapping

No custom overrides — all labels use their default names.
